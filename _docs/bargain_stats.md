# `BargainStats`

A generic key-value store for pre-computed statistical blobs that are too expensive to calculate at request time. Currently holds one record: a cross-company price comparison summary.

**Model:** `data_management/models/bargain_stats.py`

| Field | Type | Description |
|---|---|---|
| `key` | CharField (PK) | Identifier for the stat set (e.g. `'company_bargain_comparison'`) |
| `data` | JSONField | The pre-calculated payload |
| `updated_at` | DateTimeField (auto_now) | When it was last regenerated |

The model is intentionally generic — `data` is an untyped JSON blob. The shape of the data is defined by whoever writes it.

---

## The one current record: `company_bargain_comparison`

**Generated by:** `data_management/utils/generation_utils/bargain_stats_generator.py`
**Trigger:** `generate --bargain-stats`

This record contains a list of head-to-head price comparisons for every pair of companies in the DB. For each pair it answers: *"Across all products stocked by both, what percentage of the time is each company cheaper?"*

### Shape of `data`

```json
[
  {
    "company_a_name": "Coles",
    "company_b_name": "Woolworths",
    "cheaper_at_a_percentage": 38,
    "cheaper_at_b_percentage": 41,
    "same_price_percentage": 21,
    "overlap_count": 4821
  },
  ...
]
```

One entry per company pair. `overlap_count` is the number of products stocked by both companies — the sample size for the percentages.

### How it is calculated

1. Iterate all products with prices, prefetching store/company relations.
2. For each product, group prices by company. For IGA (per-store pricing), use the **average** price across IGA stores. For all others, use the **minimum** price.
3. For every company pair that both stock the product, increment `overlap_count` and award a "win" to whichever company is cheaper (or `same_price` if equal).
4. After all products are processed, convert raw win counts to percentages and write the result with `update_or_create` on key `'company_bargain_comparison'`.

### Known issue

The IGA check (`if name == 'Iga':`) uses titlecase. Every other IGA check in the codebase uses `.lower() == 'iga'`. If the company is stored as `'IGA'`, this condition never fires and IGA prices are treated as `min()` instead of `avg()`, producing incorrect stats. See recommendations.md.

## How it is served

**File:** `products/views/bargain_stats_view.py`

`GET /api/bargain-stats/` — returns `stats_object.data` directly (the JSON list). The view has a **24-hour `cache_page`**. No serializer is needed — `data` is already a plain Python list (Django deserializes JSONField on access) and DRF's `Response` renders it directly.

---

## What the frontend uses it for

The comparison percentages power the "X% of products tested were cheaper at Woolworths than Coles" text summaries on price comparison charts. These are rendered in plain HTML so search engines can read them directly.
